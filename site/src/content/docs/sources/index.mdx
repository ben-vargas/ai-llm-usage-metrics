# Data Sources

`llm-usage-metrics` supports multiple LLM coding agent data sources. Each source has a dedicated adapter that normalizes session data into a common format for aggregation.

## Supported Sources

| Source | Status | Description |
|--------|--------|-------------|
| [pi](./pi/) | ✅ Supported | Pi coding agent session data |
| [codex](./codex/) | ✅ Supported | OpenAI Codex CLI session data |
| [opencode](./opencode/) | ✅ Supported | Opencode session data |

## How Source Detection Works

The CLI automatically discovers data directories based on platform conventions:

- **macOS**: `~/Library/Application Support/{source}`
- **Linux**: `~/.config/{source}` or `~/.local/share/{source}`
- **Windows**: `%APPDATA%/{source}`

Each source adapter parses the native format and normalizes it to the internal event schema.
