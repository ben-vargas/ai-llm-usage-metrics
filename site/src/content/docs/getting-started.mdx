# Getting Started

## Installation

Install `llm-usage-metrics` globally via npm:

```bash
npm install -g llm-usage-metrics
```

Or use with npx (no installation):

```bash
npx llm-usage-metrics
```

## Requirements

- Node.js 24 or higher
- One or more supported coding agents:
  - [pi](https://pi.ai) coding agent
  - [OpenAI Codex CLI](https://github.com/openai/codex)
  - [opencode](https://github.com/opencode-ai/opencode)

## Quick Start

After installation, run the CLI to see your aggregated usage metrics:

```bash
llm-usage
```

This displays usage for the current day across all supported sources.

## First Run

On first run, the CLI will:

1. **Discover data sources** - Automatically find session data from pi, codex, and opencode
2. **Parse sessions** - Extract usage events from each source
3. **Calculate costs** - Apply LiteLLM pricing to token usage
4. **Display report** - Show formatted results in your terminal

Example output:

## Common Tasks

### View Weekly Usage

```bash
llm-usage weekly
```

### View Monthly Usage

```bash
llm-usage monthly
```

### Filter by Source

```bash
llm-usage daily --source pi
llm-usage weekly --source pi,codex
```

### Export as JSON

```bash
llm-usage daily --json > usage.json
```

### Export as Markdown

```bash
llm-usage monthly --markdown > usage-report.md
```

## Data Sources

The CLI automatically discovers data from:

| Source | macOS | Linux | Windows |
|--------|-------|-------|---------|
| **pi** | `~/Library/Application Support/pi/` | `~/.config/pi/` | `%APPDATA%/pi/` |
| **codex** | `~/.codex/` | `~/.codex/` | `%USERPROFILE%/.codex/` |
| **opencode** | `~/.opencode/` | `~/.opencode/` | `%USERPROFILE%/.opencode/` |
